Add Dante receiver

    
Maps channels of an AES67-enabled Dante device to an NMOS
    receiver. Requirements on the device: AES67 mode on, 48 kHz, PTP locked.

    
    
    

      
      
    

    

    

      
      
    

  







  

    
Add Cymatic / Archwave device

Add Lawo device (Ember+)

    
Connects over Ember+ (S101 framing, TCP). Default port is 9000.
    After adding, browse the device's tree to find the stream/routing parameters.

    
    

      
      
    

    

    

      
      
    

  







  

    
Add Dante device manually

    
For devices that discovery does not find (e.g. in another
    subnet). Legacy2NMOS queries the device directly by IP over the Dante control
    protocol (UDP 4440) — this must be routed to the device. Receivers and TX flows
    then work as usual.

    
    

    

      
      
    

  







  

    
Create multicast TX flow

    
Creates an AES67 multicast flow on  that
    transmits the selected TX channels. The device announces it via SAP, so it
    shows up as an NMOS sender. The multicast must be in the device's AES67
    range (239.<prefix>.x.x).
